Object Oriented Programming in Python, Part 1: The Basics

Object-Oriented Programming

OOP is all the rage, but the hype is real. By abstracting state to objects, we're able to keep track of data with a high degree of flexibility. The modularity of classes provides adaptability for change in the future. I came to know OOP and the Single Responsibility Principle through Ruby, and as I foray into the world of Python, I'm sharing what I learn.

Python Syntax

Below is an example of a car class in Python:


        class Car:
            number_of_wheels = 4
            def __init__(self, make, model, year, color):
                self.make = make
                self.model = model
                self.year = year
                self.color = color

            def honk(self):
                return "HONK"

            def count_wheels(self):
                return "I have %s wheels" %(self.number_of_wheels)

Below is an instantiation and example interaction:

          
            car = Car("Ford", "Escape", 2009, "white")

            print car.make
                => "Ford"

            print car.model
                => "Escape"

            print car.year
                => 2009

            print car.color
                => "white"

            print car.honk()
                => "HONK"

            print car.number_of_wheels
                => 4

            print car.count_wheels()
                => "I have 4 wheels"

In comparison to Ruby, the __init__ method is analogous to the initialize method and it takes an additional argument of self in the class construction.

Within the __init__ method, instance variables are defined as self.instance_variable and can be accessed as readers or writers. The variable number_of_wheels has been set for all instances of the car class but can be changed as shown below.

          
            car.number_of_wheels = 0
            print car.count_wheels()
                => "I have 0 wheels"

            car.make = "Chevrolet"
            print car.make
              => "Chevrolet"

Interestingly, instance methods have a different syntax from Ruby. While instance variables can be called on an instance of the class as they are in Ruby, the instance methods retain a () when called (even if the method doesn't require arguments) and take the argument of self when defined in the class. The self that is passed along is the instance of the class that the method is being called on.
